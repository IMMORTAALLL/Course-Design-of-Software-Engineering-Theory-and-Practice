# 前端 UI 设计文档

项目名称：智投社区：股票基金投资论坛系统

本文档依据 `（模块1）use_cases.md` 中的交互场景，结合 `（模块1）user_stories.md` 中的角色和用户故事，整理智投社区前端页面、路由、组件、状态和布局规范。

## 一、设计目标与角色权限

### 1. 设计目标

- 支持游客浏览公开内容，但限制发帖、评论、关注、收藏、私信等操作。
- 支持注册用户完成社区核心互动：发帖、评论、点赞、收藏、关注、私信、加入群组。
- 支持认证用户发布更专业的长文分析，并展示认证标识。
- 支持平台管理员进行板块管理、内容审核、举报处理、敏感词维护和运营统计。
- 体现股票基金社区特色，包括股票代码标签、基金标签、热门话题、风险提示和内容合规审核。

### 2. 角色与前端入口

| 角色 | 入口 | 可见页面 | 受限操作 |
| --- | --- | --- | --- |
| 游客 | 首页、板块页、帖子详情页、搜索页 | 公开内容、热门榜、用户公开主页 | 不能发帖、评论、点赞、收藏、关注、私信、加入群组 |
| 注册用户 | 前台用户端 | 发帖、评论、收藏、关注、通知、群组、个人中心 | 不能直接进入后台，不能发布认证长文 |
| 认证用户 | 前台用户端 | 注册用户页面 + 专业长文发布入口 | 不能进入后台管理 |
| 平台管理员 | 后台管理端 | 板块管理、内容审核、举报处理、敏感词管理、数据统计 | 不使用普通用户互动入口作为主要工作台 |

## 二、交互场景与页面映射

| 交互场景 | 对应页面 | 关键操作 | 系统反馈 |
| --- | --- | --- | --- |
| 游客注册成为注册用户 | 注册页 | 选择手机号/邮箱、填写验证码、提交注册 | 注册成功跳转登录页或首页；失败展示错误原因 |
| 注册用户登录系统 | 登录页 | 输入账号和密码、提交登录 | 登录成功进入首页；密码错误、账号封禁给出提示 |
| 注册用户申请成为认证用户 | 个人中心-认证申请页 | 填写认证信息、上传材料、提交审核 | 显示待审核状态；审核结果进入通知中心 |
| 游客浏览公开内容 | 首页、板块页、帖子详情页 | 浏览板块、查看帖子、搜索公开内容 | 允许浏览；触发互动时提示登录 |
| 注册用户发布普通帖子 | 发帖页 | 选择板块、填写标题正文、添加标签、上传图片/附件 | 发布成功或进入审核队列；敏感词命中时提示修改 |
| 注册用户搜索和筛选帖子 | 搜索结果页 | 输入关键词/股票代码、设置筛选条件 | 展示搜索结果；无结果时展示空状态 |
| 认证用户发布专业分析内容 | 长文发布页 | 填写分析内容、上传图表附件、确认风险提示 | 发布成功或进入审核队列 |
| 注册用户评论帖子 | 帖子详情页 | 输入评论、回复评论、提交 | 评论展示在评论区；命中敏感词时提示修改 |
| 注册用户点赞和收藏帖子 | 帖子列表、帖子详情页 | 点击点赞、收藏按钮 | 数量即时更新；重复点击取消状态 |
| 注册用户关注其他用户 | 用户公开主页 | 点击关注或取消关注 | 更新关注状态和粉丝数 |
| 注册用户加入投资主题群组 | 群组列表页、群组详情页 | 查看群组、申请加入或直接加入 | 公开群组直接加入；私密群组进入待审核 |
| 平台管理员管理论坛板块 | 后台-板块管理页 | 新增、修改、停用、删除板块 | 保存后同步影响前台板块展示 |
| 平台管理员处理可疑内容审核 | 后台-内容审核页 | 查看待审内容、通过、驳回、删除 | 审核结果通知提交用户 |
| 平台管理员处理用户举报 | 后台-举报处理页 | 查看举报对象、处理举报、处罚用户 | 保存处理记录并通知相关用户 |
| 平台管理员维护敏感词和违规规则 | 后台-敏感词管理页 | 新增、编辑、停用、删除规则 | 规则用于发布内容检测 |
| 平台管理员查看运营统计 | 后台-数据统计页 | 选择时间范围、查看图表 | 展示发帖数、评论数、活跃用户、热门标签 |

## 四、前台页面设计

### UI-001 首页

#### 页面目标

让游客和登录用户快速看到社区核心内容，包括推荐帖子、热门股票基金讨论、板块入口和搜索入口。

#### 页面结构

| 区域 | 内容 |
| --- | --- |
| 顶部导航 | Logo、全局搜索框、首页、板块、热榜、群组、登录/头像入口 |
| 左侧主内容 | 推荐信息流、精选帖子、最新帖子 |
| 右侧辅助栏 | 热门股票讨论榜、热门基金讨论榜、热门标签、风险提示 |
| 底部区域 | 平台说明、免责声明 |

#### 关键组件

- `GlobalHeader`：顶部导航和登录状态展示。
- `SearchBox`：支持关键词、股票代码、基金名称输入。
- `PostCard`：帖子卡片，展示标题、摘要、作者、认证标识、标签、评论数、点赞数。
- `HotTopicPanel`：热门话题和热门股票基金榜。
- `RiskNotice`：投资内容风险提示。

#### 交互说明

- 游客点击发帖按钮时，弹出登录提示。
- 用户点击帖子卡片进入帖子详情页。
- 点击股票代码标签进入对应标签搜索结果。
- 首页信息流优先展示已发布且非违规内容。

### UI-002 登录页

#### 页面目标

支持注册用户、认证用户和平台管理员登录系统。

#### 页面结构

| 区域 | 内容 |
| --- | --- |
| 登录表单 | 手机号/邮箱、密码、记住登录、登录按钮 |
| 辅助入口 | 注册账号、忘记密码 |
| 状态提示 | 账号不存在、密码错误、账号被封禁、账号被禁言 |

#### 交互说明

- 输入为空时禁用登录按钮或展示表单校验。
- 登录成功后普通用户进入首页，平台管理员进入后台首页。
- 被封号用户登录失败并提示账号状态异常。

### UI-003 注册页

#### 页面目标

支持游客通过手机号或邮箱注册成为注册用户。

#### 页面结构

| 区域 | 内容 |
| --- | --- |
| 注册方式切换 | 手机号注册、邮箱注册 |
| 注册表单 | 手机号/邮箱、验证码、密码、确认密码、昵称 |
| 协议区域 | 用户协议、风险提示确认 |
| 操作区域 | 注册按钮、返回登录 |

#### 交互说明

- 验证码按钮点击后进入倒计时状态。
- 密码强度不符合要求时实时提示。
- 账号已存在时提示用户直接登录。

### UI-004 板块列表页

#### 页面目标

展示所有论坛板块，帮助用户按市场、投资方法或问题类型进入讨论区。

#### 页面结构

| 区域 | 内容 |
| --- | --- |
| 市场讨论区 | A股、港股、美股、期货 |
| 主题专区 | 价值投资、量化投资、基金投资、新股新债、宏观策略 |
| 公司研究专区 | 行业讨论、个股讨论 |
| 问答求助区 | 新手提问、投资解惑 |

#### 交互说明

- 板块卡片展示板块名称、简介、帖子数、今日新增。
- 点击板块进入板块详情页。
- 停用板块在前台不展示或展示“已停用”状态。

### UI-005 板块详情页

#### 页面目标

展示某个板块下的帖子列表，并提供排序、筛选和发帖入口。

#### 页面结构

| 区域 | 内容 |
| --- | --- |
| 板块头部 | 板块名称、简介、帖子数、关注人数 |
| 筛选栏 | 最新、最热、精华、只看长文 |
| 帖子列表 | 帖子卡片列表 |
| 操作按钮 | 发帖按钮 |

#### 交互说明

- 游客点击发帖时提示登录。
- 登录用户点击发帖进入普通发帖页。
- 认证用户可选择进入长文分析发布页。

### UI-006 帖子详情页

#### 页面目标

展示完整帖子内容，并支持评论、回复、点赞、收藏、举报和关注作者。

#### 页面结构

| 区域 | 内容 |
| --- | --- |
| 帖子头部 | 标题、作者、认证标识、发布时间、板块、标签 |
| 风险提示 | “内容仅供交流，不构成投资建议” |
| 正文区域 | 文本、图片、图表、附件 |
| 互动区域 | 点赞、收藏、分享、举报 |
| 作者卡片 | 作者头像、简介、关注按钮 |
| 评论区 | 一级评论、楼中楼回复、评论输入框 |

#### 交互说明

- 游客可浏览正文和公开评论，但评论、点赞、收藏、关注需要登录。
- 评论为空时不允许提交。
- 评论命中敏感词时提示修改或进入审核。
- 举报弹窗要求选择举报原因并填写补充说明。

### UI-007 普通发帖页

#### 页面目标

支持注册用户发布普通帖子、短文讨论和投票帖。

#### 页面结构

| 区域 | 内容 |
| --- | --- |
| 基本信息 | 板块选择、帖子类型、标题 |
| 内容编辑 | 正文编辑器、图片上传、附件上传 |
| 标签设置 | 股票代码、基金名称、主题标签 |
| 投票设置 | 看涨、看跌、观望等选项 |
| 风险提示 | 合规提示和免责声明 |
| 操作区域 | 保存草稿、提交发布 |

#### 交互说明

- 标题、正文、板块为必填项。
- 上传附件限制 PDF、Excel 和图片类型。
- 命中敏感词时展示命中词和修改建议。
- 可疑内容提交后显示“等待审核”状态。

### UI-008 长文分析发布页

#### 页面目标

支持认证用户发布更专业的股票基金分析内容。

#### 页面结构

| 区域 | 内容 |
| --- | --- |
| 长文基本信息 | 标题、板块、分析类型 |
| 研究对象 | 股票代码、基金代码、行业标签 |
| 正文编辑 | 富文本或 Markdown 编辑器 |
| 材料上传 | 图表、PDF、Excel |
| 风险确认 | 风险提示确认勾选 |
| 操作区域 | 预览、提交 |

#### 交互说明

- 非认证用户访问时提示先完成认证。
- 发布前必须确认风险提示。
- 附件格式不符合要求时拒绝上传。

### UI-009 搜索结果页

#### 页面目标

支持用户按关键词、股票代码、基金名称搜索内容，并按条件筛选。

#### 页面结构

| 区域 | 内容 |
| --- | --- |
| 搜索栏 | 输入框、搜索按钮、搜索联想 |
| 筛选栏 | 时间、热度、精华、市场板块 |
| 结果列表 | 帖子、用户、标签结果 |
| 空状态 | 暂无相关内容 |

#### 交互说明

- 输入股票代码时展示代码和名称联想。
- 搜索结果高亮关键词。
- 无结果时推荐热门标签和板块。

### UI-010 用户公开主页

#### 页面目标

展示用户公开资料、认证状态、帖子、粉丝和关注关系。

#### 页面结构

| 区域 | 内容 |
| --- | --- |
| 用户头部 | 头像、昵称、认证标识、简介、关注按钮 |
| 数据概览 | 发帖数、粉丝数、关注数、影响力值 |
| 内容标签页 | 帖子、精华、收藏公开项、群组 |
| 隐私提示 | 部分信息不可见时展示提示 |

#### 交互说明

- 不能关注自己。
- 已关注用户按钮变为“取消关注”。
- 隐私限制内容不展示详情。

### UI-011 个人中心页

#### 页面目标

支持注册用户管理个人资料、收藏、关注、账号安全和认证状态。

#### 页面结构

| 区域 | 内容 |
| --- | --- |
| 个人资料 | 头像、昵称、简介、投资经验标签 |
| 投资偏好 | 关注市场、风险偏好 |
| 账号安全 | 修改密码、绑定手机/邮箱 |
| 我的内容 | 我的帖子、我的评论、我的收藏 |
| 认证状态 | 当前认证状态、申请入口 |

#### 交互说明

- 用户修改资料后实时保存或点击保存。
- 认证状态显示未认证、待审核、已认证、已驳回。

### UI-012 认证申请页

#### 页面目标

支持注册用户提交实名或专业认证申请。

#### 页面结构

| 区域 | 内容 |
| --- | --- |
| 认证类型 | 实名认证、专业认证 |
| 基本信息 | 真实姓名、证件信息、联系方式 |
| 材料上传 | 身份证明、从业资格、学历证明 |
| 提交说明 | 材料格式、审核周期 |
| 操作区域 | 提交申请、保存草稿 |

#### 交互说明

- 必填项缺失时不允许提交。
- 材料格式错误时提示重新上传。
- 提交后状态变为待审核。

### UI-013 通知中心页

#### 页面目标

集中展示评论、回复、点赞、关注、审核结果和系统通知。

#### 页面结构

| 区域 | 内容 |
| --- | --- |
| 通知分类 | 全部、互动、审核、系统 |
| 通知列表 | 通知标题、来源、时间、状态 |
| 批量操作 | 全部已读、删除 |

#### 交互说明

- 点击通知跳转到对应帖子、评论或申请记录。
- 未读通知显示明显状态。

### UI-014 收藏列表页

#### 页面目标

支持用户查看和管理收藏的帖子。

#### 页面结构

| 区域 | 内容 |
| --- | --- |
| 收藏列表 | 收藏帖子卡片 |
| 筛选条件 | 按板块、标签、收藏时间筛选 |
| 操作按钮 | 取消收藏 |

#### 交互说明

## 八、状态与异常提示设计

### 1. 游客未登录状态

游客可以浏览公开内容，但不能执行需要身份的操作。前端应在用户触发受限操作时进行轻量阻断，而不是直接跳转导致用户迷失。

| 触发操作 | 前端展示 | 后续引导 |
| --- | --- | --- |
| 点击发帖按钮 | 弹出 `LoginRequiredDialog`，提示“请先登录后再发布帖子” | 提供“去登录”“继续浏览”按钮 |
| 点击评论框 | 评论输入框置灰，提示“登录后参与讨论” | 点击后跳转登录页 |
| 点击点赞 | Toast 提示“登录后才能点赞” | 不改变点赞状态 |
| 点击收藏 | Toast 提示“登录后才能收藏帖子” | 不改变收藏状态 |
| 点击关注用户 | 弹窗提示“登录后才能关注用户” | 跳转登录页 |
| 点击私信 | 弹窗提示“登录后才能发送私信” | 跳转登录页 |
| 点击加入群组 | 弹窗提示“登录后才能加入群组” | 跳转登录页 |

未登录提示文案：

```text
当前操作需要登录后才能继续。
[去登录] [继续浏览]
```

### 2. 用户被禁言状态

禁言用户可以登录和浏览公开内容，但不能发布帖子、评论或回复。禁言状态需要在相关入口前置展示，避免用户填写完内容后才发现不能提交。

| 页面或组件 | 前端展示 |
| --- | --- |
| 首页顶部用户菜单 | 显示“账号已禁言”状态标签 |
| 普通发帖页 | 表单置灰，顶部展示禁言原因和解除时间 |
| 长文分析发布页 | 提交按钮禁用，展示禁言提示 |
| 帖子详情页评论区 | 评论框置灰，提示“你当前处于禁言状态，暂不能评论” |
| 发帖按钮 | 按钮禁用，悬停或点击展示禁言原因 |
| 通知中心 | 展示系统通知，包括禁言原因、处理时间、解除时间 |

禁言提示文案：

```text
你的账号因“发布违规荐股内容”被禁言至 2026-05-15 18:00。
禁言期间不能发帖、评论或回复，但仍可浏览公开内容。
```

### 3. 用户被封号状态

封号用户原则上不能正常使用系统。若登录时发现账号已封禁，应阻止登录；若用户已登录后被封禁，应强制退出并跳转账号异常页面。

| 场景 | 前端展示 |
| --- | --- |
| 登录时账号被封 | 登录失败，错误框提示“账号状态异常，已被封禁” |
| 已登录后被封 | 清除登录状态，跳转 `/account-banned` |
| 访问个人中心 | 跳转账号异常页 |
| 访问发帖、评论、私信 | 禁止访问，展示账号异常提示 |

账号异常页内容：

- 封号原因
- 处理时间
- 申诉说明
- 返回首页按钮

封号提示文案：

```text
账号已被封禁，原因：多次发布违规荐股或虚假信息。
如需申诉，请联系平台管理员。
```

### 4. 敏感词命中状态

敏感词命中主要发生在发帖、评论、私信、群组资料发布和后台审核场景。前端需要根据风险等级给出不同展示。

| 场景 | 前端展示 | 处理方式 |
| --- | --- | --- |
| 发帖标题命中敏感词 | 标题输入框下方红色提示 | 要求修改后再提交 |
| 发帖正文命中敏感词 | 正文中高亮命中词 | 提示修改或提交审核 |
| 评论命中敏感词 | 评论框下方展示 `SensitiveWordAlert` | 不允许直接发布 |
| 私信命中敏感词 | 弹出风险提示 | 禁止发送或进入审核 |
| 附件描述命中敏感词 | 表单字段下方提示 | 要求修改描述 |
| 长文命中风险词 | 页面顶部黄色风险提示 | 允许提交后台审核 |

风险等级展示规则：

| 风险等级 | 颜色 | 前端处理 |
| --- | --- | --- |
| 低风险 | 黄色 | 允许提交，但进入审核队列 |
| 中风险 | 橙色 | 建议修改，用户确认后可提交审核 |
| 高风险 | 红色 | 禁止直接提交，必须修改 |

敏感词提示文案：

```text
内容可能包含违规荐股或诱导交易表达，请修改后再提交。
命中词：稳赚、内幕、必涨
```

### 5. 内容审核状态

用户发布帖子、评论或附件后，内容可能进入审核流程。前端需要对作者和普通访问者展示不同视图。

| 状态 | 标签颜色 | 作者视图 | 其他用户视图 |
| --- | --- | --- | --- |
| `PENDING` 待审核 | 黄色 | 可在我的帖子中查看待审核状态 | 不可见 |
| `PUBLISHED` 已发布 | 绿色 | 正常展示 | 正常展示 |
| `REJECTED` 已驳回 | 红色 | 展示驳回原因和重新编辑入口 | 不可见 |
| `DELETED` 已删除 | 灰色 | 展示已删除状态 | 展示内容不存在 |

待审核提示文案：

```text
你的帖子已提交审核，审核通过后将在对应板块公开展示。
```

驳回提示文案：

```text
审核未通过：内容包含明确收益承诺，不符合社区规范。
请修改后重新提交。
```

### 6. 搜索和统计空状态

| 场景 | 前端展示 |
| --- | --- |
| 关键词无结果 | 展示“暂无相关内容”，推荐热门标签 |
| 股票代码无结果 | 推荐相近股票代码或热门股票讨论 |
| 基金名称无结果 | 推荐基金投资专区 |
| 高级筛选无结果 | 提供“清空筛选条件”按钮 |
| 统计暂无数据 | 图表区域展示“当前时间范围暂无统计数据” |

空状态文案：

```text
暂无相关内容。
你可以尝试更换关键词，或查看热门股票讨论。
[清空筛选条件] [查看热门标签]
```

### 7. 后台审核异常状态

| 场景 | 前端展示 |
| --- | --- |
| 内容已被其他管理员处理 | Toast 提示“该内容状态已更新，请刷新列表” |
| 驳回原因为空 | 表单校验“请填写驳回原因” |
| 敏感词重复添加 | 表单提示“该敏感词已存在” |
| 删除板块时板块下有帖子 | 二次确认弹窗，建议停用而非删除 |
| 统计计算失败 | 图表区域展示错误状态和重试按钮 |

## 九、视觉风格与布局规范

### 1. 风格定位

智投社区是投资讨论平台，界面应保持专业、清晰和可信，不使用过度娱乐化的视觉风格。

### 2. 色彩建议

| 类型 | 颜色用途 |
| --- | --- |
| 主色 | 深蓝或科技蓝，用于导航、主按钮、链接 |
| 辅助色 | 绿色表示上涨或积极，红色表示下跌或风险 |
| 中性色 | 白色、浅灰、深灰，用于背景、卡片和文本 |
| 警示色 | 橙色或红色，用于风险提示、违规提示、审核状态 |

### 3. 布局规范

- 前台采用顶部导航 + 主内容区 + 右侧信息栏布局。
- 后台采用左侧菜单 + 顶部状态栏 + 主工作区布局。
- 帖子卡片突出标题、标签、作者认证标识和互动数据。
- 表格页统一提供筛选栏、数据表格、分页和操作按钮。
- 移动端优先隐藏右侧信息栏，改为单列内容流。

### 7. 前端开发顺序

1. 先完成静态原型：四名成员分别完成自己模块页面结构，使用模拟数据，保证页面能打开、布局完整、按钮和表格存在。
2. 再抽公共组件：优先抽 `GlobalHeader`、`PostCard`、`TagSelector`、`RiskNotice`、`EmptyState`、`ConfirmDialog`、`AuditStatusTag`。
3. 再进行接口联调：按登录注册、首页板块、发帖、帖子详情、评论点赞收藏、后台审核的业务闭环推进。
4. 最后补数据可视化：实现后台首页统计卡片、数据统计页 ECharts 图表、热门股票/基金榜单。

## 十二、最终采用方案

最终 UI 设计采用“前台社区 + 后台管理”的双端结构：

- 前台围绕游客浏览、用户发帖、内容互动、关注群组和搜索推荐展开。
- 后台围绕板块管理、内容审核、举报处理、敏感词规则和运营统计展开。
- 页面设计保留股票基金社区特征，重点展示股票/基金标签、认证标识、风险提示、热门话题和合规审核。
- 课程实现阶段优先完成高优先级页面，低优先级功能可做简化实现或状态模拟。
- 前端开发阶段按 `auth`、`forum`、`interaction`、`admin`、`shared` 拆分目录和任务，使 UI 设计能够直接指导后续 Vue 项目编码。
