# 部署说明

项目名称：智投社区：股票基金投资论坛系统

本文档说明课程设计验收采用的本地单机部署方式。当前项目按前后端分离运行，后端使用 FastAPI + SQLAlchemy，前端使用 Vue 3 + Vite，数据库默认使用本地 SQLite。

## 一、部署目标

本地部署目标如下：

| 项目 | 部署方式 |
| --- | --- |
| 后端服务 | Uvicorn 启动 FastAPI，监听 `127.0.0.1:8000` |
| 前端服务 | Vite 开发服务器，监听 `127.0.0.1:5173` |
| 数据库 | SQLite，本地文件 `backend/forum_system.db` |
| 接口文档 | FastAPI 自动生成，访问 `http://127.0.0.1:8000/docs` |
| 前端访问 | 浏览器访问 `http://127.0.0.1:5173` |

该方式不依赖云服务器，适合课程演示、功能验收和本机复现。

## 二、环境要求

| 环境 | 要求 |
| --- | --- |
| 操作系统 | Windows 10/11，macOS 或 Linux 均可 |
| Python | 推荐 Python 3.10+ |
| Node.js | 推荐 Node.js 18+ |
| 包管理工具 | `pip`、`npm` |
| 浏览器 | Chrome、Edge 或其他现代浏览器 |

如果 Windows 默认 `python` 指向旧版本，可以使用 `py -3.10`、`py -3.11`、`py -3.12` 或已安装依赖的 Python 解释器。

## 三、后端部署

从项目根目录进入后端目录：

```powershell
cd backend
```

安装依赖：

```powershell
python -m pip install -r requirements.txt
```

启动后端：

```powershell
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

启动成功后检查：

| 地址 | 用途 |
| --- | --- |
| `http://127.0.0.1:8000/api/health` | 健康检查 |
| `http://127.0.0.1:8000/docs` | Swagger 接口文档 |

首次启动时，系统会自动创建 `backend/forum_system.db`，并写入用户、板块、帖子、评论、群组、通知、审核和举报等演示数据。

## 四、使用项目内依赖目录启动

如果本机已经按测试流程把后端依赖安装到 `backend/venv_lib`，可以在 `backend` 目录下使用以下方式启动：

```powershell
$env:PYTHONPATH = (Resolve-Path '.\venv_lib').Path + ';' + (Resolve-Path '.').Path
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

该方式适合在不污染系统 Python 环境的情况下复现后端运行。

## 五、前端部署

打开新的终端，从项目根目录进入前端目录：

```powershell
cd frontend
```

安装依赖：

```powershell
npm install
```

启动前端：

```powershell
npm run dev -- --host 127.0.0.1 --port 5173
```

启动成功后访问：

```text
http://127.0.0.1:5173
```

Vite 已配置代理，会将 `/api` 请求转发到 `http://127.0.0.1:8000`。如果后端使用其他端口，可以在启动前设置：

```powershell
$env:VITE_API_PROXY_TARGET = "http://127.0.0.1:8001"
npm run dev
```

## 七、部署验证

后端编译检查：

```powershell
python -m compileall backend\app
```

后端接口烟测：

```powershell
$env:PYTHONPATH = (Resolve-Path 'backend\venv_lib').Path + ';' + (Resolve-Path 'backend').Path
python scripts\backend_smoke_test.py
```

前端构建检查：

```powershell
npm run build
```

该命令需要在 `frontend` 目录下执行。构建通过后，说明 TypeScript 类型检查和 Vite 打包均可完成。

## 八、常见问题

| 问题 | 处理方式 |
| --- | --- |
| `No module named fastapi` | 后端依赖未安装，进入 `backend` 后执行 `python -m pip install -r requirements.txt` |
| 端口 `8000` 被占用 | 关闭占用该端口的进程，或临时改用其他端口并同步设置前端代理 |
| 端口 `5173` 被占用 | Vite 可能自动切换端口，也可以手动指定 `--port 5174` |
| 前端请求失败 | 确认后端已启动，并检查 `VITE_API_PROXY_TARGET` 是否指向正确后端地址 |
| 登录失败 | 确认使用演示账号和密码 `Admin123456`，并确认后端数据库已初始化 |
| 数据库异常 | 先备份当前 `backend/forum_system.db`，再由小组确认是否重新初始化本地演示库 |

## 九、可选 MySQL 部署

课程验收默认使用 SQLite。若后续需要切换 MySQL，可按以下思路处理：

1. 在 MySQL 中创建数据库。
2. 复制 `backend/.env.example` 为 `backend/.env`。
3. 修改 `.env` 中的 `DATABASE_URL`。
4. 导入 `database/schema.sql` 和 `database/seed.sql`。
5. 重启后端服务。

本次课程设计交付采用 SQLite 本地部署，不要求必须切换 MySQL。
