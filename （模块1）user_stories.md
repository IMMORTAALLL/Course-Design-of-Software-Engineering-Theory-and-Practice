# 用户故事

指导书要求：使用提示词：“作为产品经理，为【***系统】中如下功能生成用户故事：【***功能描述】。要求用户故事遵循以下格式：作为一名…我想…以便...。”

格式：作为一名【角色】，我想要【能力】，以便【价值】。

优先级说明：

- 高：核心业务闭环，必须优先实现。
- 中：重要功能，建议在核心功能完成后实现。
- 低：扩展功能，可根据进度选择实现或做模拟实现。
