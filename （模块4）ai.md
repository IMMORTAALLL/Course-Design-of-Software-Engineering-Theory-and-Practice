# AI 使用记录

## 模块四：AI辅助测试与调试

### 一、仓库现状核对

#### 原始提示词

```text
拉取一下最新的仓库，分析一下整个项目的现状。现在接管ABCD全部任务，详细分析现状，禁止虚构。
```

#### AI输出摘要

AI 先执行 Git 检查和同步命令，确认当前本地仓库在 `docx` 分支，`main`、`origin/main`、`origin/docx` 位于同一提交，远端无未拉取内容。随后读取 README、模块二后端接口文档、模块二 UI 设计文档、后端路由和前端路由，整理出 ABCD 四个模块的真实实现状态。

#### 人工检查

检查过程中发现仓库中的 `HANDOFF.md` 是旧阶段交接材料，其中分支、端口和未提交状态已经过时。因此不能直接按旧交接文档判断当前项目状态，必须以当前 Git 输出和源码为准。

#### 迭代优化

后续测试和修复均基于当前 `docx` 分支实际代码，不合并过时分支，不使用未验证的历史结论。


### 二、后端接口测试

#### 原始提示词

```text
进行全面专业测试。先检查后端编译，再使用临时数据库测试核心接口，避免污染仓库。
```

#### AI输出摘要

AI 首先执行：

```bash
python -m compileall backend\app
```

后端编译通过。随后尝试用系统默认 Python 运行接口测试，但发现默认 Python 3.8.6 加载的 SQLAlchemy 不支持项目使用的 `DeclarativeBase`。

#### 人工检查

该问题属于本机测试环境问题，不是业务代码问题。不能把第一次失败误写成接口失败，也不能跳过接口测试。

#### 迭代优化

AI 改用 Python 3.12 临时测试环境执行 FastAPI TestClient 回归测试，并通过环境变量指定临时 SQLite 数据库。最终完成 21 个后端接口与业务流程检查，结果为 21 项通过、0 项失败。


### 三、功能缺口修复后的验证

#### 原始提示词

```text
对照项目要求检查 A/B/C/D，发现问题就修复，修复后再测试。
```

#### AI输出摘要

AI 在源码检查中发现两个主要问题：

- B 模块发帖接口固定 `user_id=1`，没有使用当前登录用户。
- C 模块社交互动目录基本为空，缺少评论、点赞、收藏、关注、通知和群组功能。

AI 随后补充：

- `POST /api/posts` 使用当前登录用户创建帖子。
- `POST /api/posts/analysis` 支持专业认证用户发布长文分析。
- 新增社交互动后端模型、服务和路由。
- 新增社交互动前端 API、类型、组件和页面。
- 帖子详情、用户公开主页、个人中心和顶部导航接入互动功能。

#### 人工检查

实现时要求不能虚构功能完成情况，因此每个新增功能都需要有后端接口、前端入口或测试覆盖作为依据。对于没有做完的扩展项，例如群内资料区，只保留合理空状态，不写成已实现。

#### 迭代优化

修复后再次执行后端接口回归和前端构建检查，确认新增功能没有破坏原有 A/B/D 模块。


### 四、前端构建测试

#### 原始提示词

```text
前端也要做全面检查，确认新增页面、路由、组件能正常构建。
```

#### AI输出摘要

AI 执行：

```bash
cd frontend
npm run build
```

构建过程包含 `vue-tsc --noEmit` 和 `vite build`。最终构建通过，说明新增的通知中心、收藏列表、关注动态、群组页面、评论组件、关注按钮、点赞收藏按钮等均通过 TypeScript 和 Vite 检查。

#### 人工检查

前端实际项目没有使用 Element Plus，尽管模块二设计文档中技术栈写了 Element Plus。因此新增界面没有强行引入新 UI 库，而是沿用当前项目已有的原生 Vue + CSS 风格。

#### 迭代优化

将新增页面统一使用项目已有 `.page`、`.panel`、`.stack`、`.primary-button` 等样式，并把互动入口放入顶部导航和个人中心，保证用户可以从现有流程进入新增功能。

