# 用例交互场景

每个用例建议包含：参与者、前置条件、主流程、异常流程、后置条件。

## 用户与权限模块

### 场景1：游客注册成为注册用户

- 参与者：游客
- 前置条件：游客未登录系统，进入智投社区注册页面。
- 主流程：
  1. 游客选择手机号或邮箱注册方式。
  2. 系统展示注册表单，要求填写账号、密码、昵称和验证码。
  3. 游客填写注册信息并提交。
  4. 系统校验账号格式、密码强度和验证码。
  5. 系统创建用户账号，并将用户身份标记为注册用户。
  6. 系统跳转到登录页面或自动进入首页。
- 异常流程：
  1. 如果账号已存在，系统提示“该账号已注册”。
  2. 如果验证码错误或过期，系统提示重新获取验证码。
  3. 如果密码不符合要求，系统提示用户修改密码。
- 后置条件：游客完成注册，成为注册用户，可进行发帖、评论、关注、收藏、私信和加入群组等操作。

### 场景2：注册用户登录系统

- 参与者：注册用户
- 前置条件：注册用户已完成账号注册，当前未登录。
- 主流程：
  1. 注册用户进入登录页面。
  2. 注册用户输入手机号或邮箱、密码。
  3. 系统校验账号和密码是否正确。
  4. 系统生成登录状态，并进入智投社区首页。
  5. 系统根据用户身份展示可用功能入口。
- 异常流程：
  1. 如果账号不存在，系统提示用户先注册。
  2. 如果密码错误，系统提示重新输入。
  3. 如果用户已被封号，系统提示账号状态异常，禁止登录或限制操作。
- 后置条件：注册用户登录成功，可以使用注册用户权限范围内的功能。

### 场景3：注册用户申请成为认证用户

- 参与者：注册用户
- 前置条件：注册用户已登录系统，账号状态正常。
- 主流程：
  1. 注册用户进入个人中心的认证申请页面。
  2. 系统展示实名认证或专业认证所需填写的信息。
  3. 注册用户填写真实姓名、认证类型、证明材料等信息。
  4. 注册用户提交认证申请。
  5. 系统保存认证申请，并将申请状态标记为待审核。
  6. 平台管理员在后台审核通过后，系统将用户身份更新为认证用户。
- 异常流程：
  1. 如果必填信息缺失，系统提示补全后再提交。
  2. 如果材料格式不符合要求，系统提示重新上传。
  3. 如果审核不通过，系统记录原因并通知用户重新提交。
- 后置条件：认证通过后，注册用户成为认证用户，可展示认证标识并获得更高可信度。

## 论坛内容模块

### 场景1：游客浏览公开内容

- 参与者：游客
- 前置条件：游客未登录系统，访问智投社区首页。
- 主流程：
  1. 游客进入首页，查看公开板块列表。
  2. 游客选择感兴趣的板块，例如 A股讨论区、基金投资专区或问答求助区。
  3. 系统展示该板块下的公开帖子列表。
  4. 游客点击帖子标题，查看帖子详情、图片、标签和公开评论。
  5. 游客可通过关键词搜索公开帖子。
- 异常流程：
  1. 如果帖子已被删除或下架，系统提示内容不存在。
  2. 如果游客尝试发帖、评论、关注或私信，系统提示先登录。
- 后置条件：游客只能浏览公开内容，不能进行发帖、评论、关注或私信操作。

### 场景2：注册用户发布普通帖子

- 参与者：注册用户
- 前置条件：注册用户已登录，账号未被禁言或封号。
- 主流程：
  1. 注册用户进入发帖页面。
  2. 注册用户选择帖子所属板块。
  3. 注册用户填写标题、正文、股票或基金标签。
  4. 注册用户可选择上传图片或附件。
  5. 注册用户提交帖子。
  6. 系统进行基础校验和敏感词检查。
  7. 系统发布帖子，或将可疑帖子放入审核队列。
- 异常流程：
  1. 如果标题或正文为空，系统提示补全内容。
  2. 如果用户被禁言，系统禁止发布并提示禁言原因。
  3. 如果内容命中敏感词，系统提示修改或提交后台审核。
- 后置条件：帖子发布成功后进入对应板块；如需审核，则等待平台管理员处理。

### 场景3：注册用户搜索和筛选帖子

- 参与者：注册用户
- 前置条件：注册用户已登录，系统中存在公开帖子。
- 主流程：
  1. 注册用户在搜索框输入关键词、股票代码或股票名称。
  2. 系统根据标题、正文和标签进行匹配。
  3. 注册用户选择按时间、热度、精华或市场板块筛选。
  4. 系统展示符合条件的帖子列表。
  5. 注册用户点击帖子进入详情页。
- 异常流程：
  1. 如果没有匹配结果，系统提示暂无相关内容。
  2. 如果关键词格式不合法，系统提示重新输入。
- 后置条件：注册用户获得符合条件的帖子结果，可继续浏览、评论、收藏或关注作者。

### 场景4：认证用户发布专业分析内容

- 参与者：认证用户
- 前置条件：认证用户已登录，账号状态正常。
- 主流程：
  1. 认证用户进入长文分析发布页面。
  2. 认证用户选择板块和内容类型。
  3. 认证用户填写分析标题、正文、股票或基金标签。
  4. 认证用户上传图表或附件材料。
  5. 系统展示风险提示，提醒内容仅供交流参考。
  6. 认证用户确认后提交。
  7. 系统发布内容或进入后台审核。
- 异常流程：
  1. 如果内容包含违规荐股表达，系统提示修改或提交审核。
  2. 如果附件格式不符合要求，系统拒绝上传。
- 后置条件：专业分析内容进入论坛内容列表，认证用户标识随内容展示。

## 社交互动模块

### 场景1：注册用户评论帖子

- 参与者：注册用户
- 前置条件：注册用户已登录，帖子处于可评论状态。
- 主流程：
  1. 注册用户进入帖子详情页。
  2. 注册用户在评论框输入评论内容。
  3. 注册用户提交评论。
  4. 系统检查评论内容是否为空或包含敏感词。
  5. 系统保存评论并展示在评论区。
  6. 系统向帖子作者发送评论通知。
- 异常流程：
  1. 如果用户未登录，系统提示登录后评论。
  2. 如果用户被禁言，系统禁止评论。
  3. 如果评论命中敏感词，系统提示修改或进入审核。
- 后置条件：评论发布成功，帖子评论数更新。

### 场景2：注册用户点赞和收藏帖子

- 参与者：注册用户
- 前置条件：注册用户已登录，帖子存在且可见。
- 主流程：
  1. 注册用户进入帖子详情页或帖子列表。
  2. 注册用户点击点赞按钮。
  3. 系统记录点赞关系并更新点赞数。
  4. 注册用户点击收藏按钮。
  5. 系统记录收藏关系，并将帖子加入用户收藏列表。
- 异常流程：
  1. 如果重复点赞，系统取消点赞或提示已点赞。
  2. 如果帖子被删除，系统提示内容不可操作。
  3. 如果用户未登录，系统提示登录后操作。
- 后置条件：帖子互动数据更新，注册用户可在个人中心查看收藏内容。

### 场景3：注册用户关注其他用户

- 参与者：注册用户
- 前置条件：注册用户已登录，被关注用户存在。
- 主流程：
  1. 注册用户进入其他用户主页。
  2. 注册用户点击关注按钮。
  3. 系统保存关注关系。
  4. 系统更新双方关注数和粉丝数。
  5. 被关注用户发布新内容时，系统可在关注动态中展示。
- 异常流程：
  1. 如果用户关注自己，系统禁止操作。
  2. 如果已关注该用户，系统提示已关注或执行取消关注。
  3. 如果被关注用户不存在，系统提示用户不存在。
- 后置条件：注册用户可在关注列表中查看被关注用户动态。

### 场景4：注册用户加入投资主题群组

- 参与者：注册用户
- 前置条件：注册用户已登录，目标群组存在。
- 主流程：
  1. 注册用户进入群组列表页面。
  2. 注册用户查看公开群组或私密群组信息。
  3. 注册用户点击加入群组。
  4. 如果是公开群组，系统直接加入。
  5. 如果是私密群组，系统生成加入申请，等待审核。
- 异常流程：
  1. 如果用户已加入该群组，系统提示无需重复加入。
  2. 如果群组已关闭，系统提示无法加入。
- 后置条件：注册用户加入群组后，可查看群内专属讨论和资料分享。

## 审核与后台模块

### 场景1：平台管理员管理论坛板块

- 参与者：平台管理员
- 前置条件：平台管理员已登录后台管理系统。
- 主流程：
  1. 平台管理员进入板块管理页面。
  2. 系统展示现有板块列表及状态。
  3. 平台管理员新增、修改或删除板块。
  4. 系统校验板块名称、排序和状态。
  5. 系统保存变更，并更新前台板块展示。
- 异常流程：
  1. 如果板块名称为空或重复，系统提示修改。
  2. 如果板块下仍有帖子，删除时系统提示确认或改为停用。
  3. 如果当前账号没有后台权限，系统拒绝访问。
- 后置条件：论坛板块信息更新，游客、注册用户和认证用户在前台看到最新板块。

### 场景2：平台管理员处理可疑内容审核

- 参与者：平台管理员
- 前置条件：系统中存在待审核帖子、评论或附件。
- 主流程：
  1. 平台管理员进入内容审核队列。
  2. 系统展示待审核内容、提交用户、命中原因和提交时间。
  3. 平台管理员查看内容详情。
  4. 平台管理员选择通过、驳回、删除或要求修改。
  5. 系统保存审核结果，并通知内容提交用户。
- 异常流程：
  1. 如果内容已被其他管理员处理，系统提示状态已更新。
  2. 如果审核意见为空且选择驳回，系统提示填写原因。
- 后置条件：审核结果生效，内容按结果展示、下架或退回修改。

### 场景3：平台管理员处理用户举报

- 参与者：平台管理员
- 前置条件：注册用户或认证用户已提交举报记录。
- 主流程：
  1. 平台管理员进入举报管理页面。
  2. 系统展示举报对象、举报原因、举报人和举报时间。
  3. 平台管理员查看被举报内容和用户历史记录。
  4. 平台管理员判断举报是否成立。
  5. 平台管理员选择驳回举报、删除内容、警告用户、禁言用户或封号用户。
  6. 系统记录处理结果，并通知相关用户。
- 异常流程：
  1. 如果举报对象已删除，系统提示内容不存在并允许关闭举报。
  2. 如果证据不足，平台管理员可标记为不成立。
- 后置条件：举报处理完成，后台保留处理记录。

### 场景4：平台管理员维护敏感词和违规规则

- 参与者：平台管理员
- 前置条件：平台管理员已登录后台管理系统。
- 主流程：
  1. 平台管理员进入敏感词管理页面。
  2. 系统展示敏感词列表、风险等级和启用状态。
  3. 平台管理员新增、修改、停用或删除敏感词。
  4. 系统保存敏感词规则。
  5. 用户发布内容时，系统根据规则进行命中检测。
- 异常流程：
  1. 如果敏感词重复，系统提示已存在。
  2. 如果规则为空，系统禁止保存。
- 后置条件：敏感词规则生效，系统可用于内容过滤、风险提示和审核队列生成。

### 场景5：平台管理员查看运营统计

- 参与者：平台管理员
- 前置条件：系统中已产生帖子、评论、点赞、收藏、关注等数据。
- 主流程：
  1. 平台管理员进入数据统计页面。
  2. 系统展示论坛活跃度统计，包括发帖数、评论数和活跃用户数。
  3. 系统展示热门话题分析，包括热门标签和关键词。
  4. 系统展示用户参与度报告，包括评论、点赞、收藏和关注数据。
  5. 平台管理员根据统计结果调整内容运营策略。
- 异常流程：
  1. 如果统计时间范围内没有数据，系统提示暂无统计结果。
  2. 如果统计计算失败，系统提示稍后重试。
- 后置条件：平台管理员掌握论坛运营情况，可用于精选内容、审核管理和推荐运营。
