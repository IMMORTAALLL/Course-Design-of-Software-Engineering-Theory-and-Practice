# AI 使用记录

## 模块五：本地部署与交付文档

### 时间：2026.6.23

## 一、使用目标

本模块使用 AI 辅助整理本地部署说明、用户手册、项目总结报告和工作完成情况记录。人工核对重点包括：

- 部署方式必须符合当前项目实际运行方式。
- 不引入云服务器、Docker 或外部平台作为必需条件。
- 后端端口、前端端口、数据库位置和演示账号必须与当前代码一致。
- 用户手册必须覆盖普通用户、专业用户和管理员的主要操作。
- 文档表述不能替代真实验证，仍需通过后端编译、后端烟测和前端构建确认。

## 二、关键提示词记录

### 1. 部署方式确认

提示词摘要：

```text
根据当前 FastAPI + Vue 3 + Vite 项目结构，按本地单机部署方式编写课程设计部署说明。
要求使用本地 SQLite，不使用云服务器，不引入 Docker。
说明后端、前端、数据库、演示账号、验证命令和常见问题。
```

采用结果：

- 后端使用 Uvicorn 启动，监听 `127.0.0.1:8000`。
- 前端使用 Vite 启动，监听 `127.0.0.1:5173`。
- 数据库使用 `backend/forum_system.db`。
- 保留 MySQL 作为可选部署说明，不作为本次验收必需项。

### 2. 用户手册生成

提示词摘要：

```text
为股票基金投资论坛系统编写用户手册。
覆盖未登录浏览、登录注册、个人中心、发帖、搜索、热榜、评论、点赞、收藏、关注、私信、群组和后台管理。
```

采用结果：

- 按用户访问路径组织文档。
- 区分普通用户、专业用户和管理员。
- 将后台管理单独列出，说明普通用户不能访问后台。

### 3. 项目总结报告整理

提示词摘要：

```text
结合模块一到模块五成果，编写课程设计项目总结报告。
需要列出需求分析、系统设计、编码实现、测试调试、部署交付的成果和对应文档。
```

采用结果：

- 按模块列出成果和文件。
- 汇总用户与权限、论坛内容、社交互动、群组交流、后台管理和数据库功能。
- 总结本地部署方式和后续改进方向。

## 三、人工核对与调整

| 核对项 | 调整结果 |
| --- | --- |
| 部署方式 | 保持本地单机部署，不写成云部署 |
| 数据库 | 明确 SQLite 为默认验收数据库，MySQL 只作为可选项 |
| 端口 | 后端固定写 `8000`，前端固定写 `5173` |
| 账号 | 使用 `admin@stockforum.com`、`value@stockforum.com`、`quant@stockforum.com` |
| 前端代理 | 写明 Vite 将 `/api` 转发到后端 |
| 验证命令 | 使用项目已有编译、烟测和构建命令 |
| 分工顺序 | 文档中将负责人谢亨义放在第一位 |
