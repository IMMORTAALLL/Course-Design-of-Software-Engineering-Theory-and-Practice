# 测试报告

项目名称：智投社区：股票基金投资论坛系统

本文档记录模块四“AI辅助测试与调试”的测试过程。测试对象为模块三已经完成的前后端代码，重点检查 FastAPI 后端接口、Vue 3 前端页面、后台审核管理模块以及前后端联调流程是否能够正常运行。

## 一、测试目标

本次测试的目标如下：

- 检查后端项目代码能否正常编译和导入。
- 检查前端 TypeScript 类型检查和 Vite 构建是否通过。
- 检查核心接口是否按照统一响应格式返回数据。
- 检查后台管理页面能否正常加载审核、举报、敏感词、用户处置和统计数据。
- 检查后台审核、举报处理、敏感词启停等按钮操作是否能正确调用后端接口。
- 记录测试过程中发现的环境问题和处理方式，为后续部署和答辩演示提供依据。

## 五、发现的问题与处理

### 1. Python 版本不匹配

测试时发现系统默认 Python 版本为 3.13.2，而仓库 `backend/venv_lib` 中的 `pydantic_core` 二进制文件为 Python 3.12 版本，直接运行后端接口测试会出现导入失败。

处理方式：使用 Python 3.12 环境运行后端服务，并通过 `PYTHONPATH=.;venv_lib` 加载仓库中的依赖。后续部署或答辩演示时，需要保持 Python 版本和依赖二进制版本一致。

### 2. TestClient 缺少依赖

测试过程中尝试使用 FastAPI 的 `TestClient` 做接口测试，但当前依赖中没有安装 `httpx`，因此 `TestClient` 无法使用。

处理方式：没有修改项目依赖，而是改用 Uvicorn 启动真实后端服务，再通过 HTTP 请求完成接口冒烟测试。该方式更接近真实运行环境。

### 3. 测试数据库隔离

为了避免测试影响仓库中的本地数据库文件，本次测试使用临时 SQLite 数据库。后台演示数据由后端启动时自动写入，测试结束后临时数据不会作为项目文件提交。
